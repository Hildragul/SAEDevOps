Room : Metasploit Introduction

Task 1 – Introduction to Metasploit

Cette room introduit Metasploit, un outil incontournable en pentest. Il est utilisé aussi bien pour la recherche de vulnérabilités que pour l’exploitation, et même les étapes après compromission.

Il existe deux versions :

  • Metasploit Pro (version payante avec interface graphique)
  • Metasploit Framework (version open-source en ligne de commande)

Avec Metasploit Framework, on peut :

  • Scanner et collecter des infos
  • Exploiter des failles
  • Gérer des payloads (code injecté sur la cible)
  • Automatiser certains tests

Les composants principaux sont :

  • msfconsole → interface principale en ligne de commande
  • Modules → exploits, scanners, payloads, etc.
  • Outils → comme msfvenom (création de payloads), pattern_create, etc.

L’objectif de cette room est de te rendre à l’aise avec l’environnement Metasploit, pour savoir :

  • Trouver un exploit
  • Configurer les bons paramètres
  • Lancer l’attaque contre un service vulnérable

Task 2 – Main Components of Metasploit

Résumé : On découvre les types de modules et les notions clés comme vulnérabilité, exploit, payload.

Notions clés :

  • Vulnérabilité : faille dans un système
  • Exploit : code qui exploite une vulnérabilité
  • Payload : code exécuté sur la cible

Types de modules :

  • Exploit
  • Payload
  • Auxiliary
  • Post
  • Encoders
  • Evasion
  • NOPs

Types de payload : singles (autonomes), stagers + stages (en deux parties)

Trick : "_" = single ; "/" = staged

Questions et réponses :

  • Q1 : What is the name of the code taking advantage of a flaw on the target system? R : Exploit
  • Q2 : What is the name of the code that runs on the target system to achieve the attacker's goal? R : Payload
  • Q3 : What are self-contained payloads called? R : Singles
  • Q4 : Staged or single? R : Singles

Task 3 – Msfconsole

Résumé : msfconsole est l’interface principale du framework Metasploit. Elle permet de :

  • Lancer des modules avec use
  • Afficher les options avec show options
  • Rechercher avec search
  • Afficher des infos avec info
  • Auto-compléter avec Tab
  • Utiliser l’historique
  • Revenir avec back

Exemple d’utilisation :

use exploit/windows/smb/ms17_010_eternalblue

Autres commandes utiles :

  • show payloads
  • search type:auxiliary telnet
  • Rang des exploits : great, normal, manual…

Questions et réponses :

  • Q1 : How would you search for a module related to Apache? R : search apache
  • Q2 : Who provided the auxiliary/scanner/ssh/ssh_login module? R : todb

Task 4 – Working with Modules

Résumé : Chaque module a ses propres options à configurer. Les commandes principales :

  • use [module]
  • show options
  • set [paramètre] [valeur]
  • setg [paramètre] [valeur]
  • unset [paramètre] ou unset all
  • exploit ou run
  • sessions et sessions -i [id]
  • background ou CTRL+Z

Réponses aux questions :

  • Q1: How would you set the LPORT value to 6666? R : set LPORT 6666
  • Q2: How would you set the global value for RHOSTS to 10.10.19.23? R : setg RHOSTS 10.10.19.23
  • Q3: What command would you use to clear a set payload? R : unset payload
  • Q4: What command do you use to proceed with the exploitation phase? R : exploit
