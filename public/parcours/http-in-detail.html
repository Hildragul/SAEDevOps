Room : HTTP in Detail

Contexte général de la room

Cette room explore en profondeur le protocole HTTP(S), la base de la communication web. Elle a pour objectif de comprendre comment les navigateurs web et les serveurs échangent les données, et ce qui différencie HTTP de sa version sécurisée, HTTPS. C’est une étape essentielle pour toute personne qui souhaite comprendre la sécurité et le fonctionnement des sites web.

Task 1 – What is HTTP(S)?

Résumé de la tâche :

  • HTTP (HyperText Transfer Protocol) est le protocole utilisé pour accéder à du contenu sur le web : pages HTML, images, vidéos, etc.
  • HTTPS (HyperText Transfer Protocol Secure) est la version sécurisée de HTTP, qui chiffre les échanges de données pour assurer confidentialité, intégrité et authenticité.

Questions et réponses :

  • Q1 : What does HTTP stand for? R : HyperText Transfer Protocol
  • Q2 : What does the S in HTTPS stand for? R : Secure
  • Q3 : What is the challenge flag? R : THM{INVALID_HTTP_CERT}

Task 2 – Requests and Responses

Résumé de la tâche :

Partie 1 : La structure d’une URL

  • Scheme : le protocole utilisé
  • User:Password : identifiants
  • Host : nom de domaine ou IP
  • Port : port du serveur
  • Path : chemin vers la ressource
  • Query string : paramètres
  • Fragment : partie spécifique de la page

Partie 2 : Requête HTTP

GET / HTTP/1.1
Host: tryhackme.com
User-Agent: Mozilla/5.0 Firefox/87.0
Referer: https://tryhackme.com/

Partie 3 : Réponse HTTP

HTTP/1.1 200 OK
Server: nginx/1.15.8
Date: Fri, 09 Apr 2021 13:34:03 GMT
Content-Type: text/html
Content-Length: 98

Questions et réponses :

  • Q1 : What HTTP protocol is being used in the above example? R : HTTP/1.1
  • Q2 : What response header tells the browser how much data to expect? R : Content-Length

Task 3 – HTTP Methods

Résumé de la tâche :

  • GET : récupérer des informations
  • POST : envoyer des données (ex. créer un compte)
  • PUT : modifier des infos existantes
  • DELETE : supprimer une ressource

Questions et réponses :

  • Q1 : What method would be used to create a new user account? R : POST
  • Q2 : What method would be used to update your email address? R : PUT
  • Q3 : What method would be used to remove a picture you've uploaded to your account? R : DELETE
  • Q4 : What method would be used to view a news article? R : GET

Task 4 – HTTP Status Codes

Résumé de la tâche :

  • 100-199 : Info
  • 200-299 : Succès
  • 300-399 : Redirections
  • 400-499 : Erreurs client
  • 500-599 : Erreurs serveur

Questions et réponses :

  • Q1 : New user or blog post article? R : 201
  • Q2 : Page not found? R : 404
  • Q3 : App crash, server issue? R : 500
  • Q4 : Editing profile without login? R : 401

Task 5 – Headers

Résumé de la tâche :

Requête (client → serveur) :

  • Host
  • User-Agent
  • Content-Length
  • Accept-Encoding
  • Cookie

Réponse (serveur → client) :

  • Set-Cookie
  • Cache-Control
  • Content-Type
  • Content-Encoding

Questions et réponses :

  • Q1 : What header tells the web server what browser is being used? R : User-Agent
  • Q2 : What header tells the browser what type of data is being returned? R : Content-Type
  • Q3 : What header tells the web server which website is being requested? R : Host

Task 6 – Cookies

Résumé de la tâche :

  • Les cookies permettent de conserver des infos entre les requêtes (stateless)
  • Set-Cookie → pour les envoyer au client
  • Cookie → renvoyé au serveur à chaque requête

Question et réponse :

  • Q : Which header is used to save cookies to your computer? R : Set-Cookie
